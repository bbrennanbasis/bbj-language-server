### Integration Findings

All cross-phase wiring is complete. Key integration paths verified:

- **Java DTO -> Langium types -> Scope/Completion:** `InteropService.java` populates `isStatic`/`isDeprecated`/`constructors` -> `java-interop.ts` Phase 1 sync maps to Langium types -> `bbj-scope.ts` filters static methods -> `bbj-completion-provider.ts` applies deprecated tags and constructor items
- **Token builder -> Grammar -> AST:** `EXIT_NO_NL` token built in `bbj-token-builder.ts` -> spliced into vocabulary -> grammar rule in `bbj.langium` -> `ExitWithNumberStatement` in generated AST
- **Grammar -> Validator -> Recovery:** `VariableDecl` added to `ClassMember` grammar -> `checkDeclareNotInClassBody` validator registered -> linker cast updated for expanded union type

### Known Limitation (Not Blocker)

**FEAT-02 FQN path:** `java.lang.String.` via MemberCall chain does not filter to static-only methods. The `isClassRef` detection in `bbj-scope.ts` only fires for `SymbolRef` receivers (USE aliases), not `MemberCall` chains. Documented in 59-04-SUMMARY.md — requires JAR redeployment to enable safely.
